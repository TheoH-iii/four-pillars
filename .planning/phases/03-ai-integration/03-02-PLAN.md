Add two new global functions to app.js. Place them after the existing `dataRows()` helper and before the card builder functions.

**buildChartSummary(chart, cardType)**
Extracts a compact summary object from `window.currentChart` to send to the API. Returns:
```js
{
  pillars,        // "甲子 · 丙寅 · 戊午 · 庚申"
  dayMaster,      // eightChar.getDayGan()
  dayMasterElement, // STEM_ELEMENT[dayMaster] mapped to English name via ELEMENT_NAMES
  strength,       // analysis.strength ("strong" or "weak")
  pattern,        // analysis.pattern.en
  favorableElements, // analysis.favorable array mapped to English names
  cardData        // card-specific slice — see below
}
```

cardData slice per cardType:
- "overview": `{ stars: analysis.stars, tenDeities: top 3 from getDeityDistribution() }`
- "fiveElements": `{ distribution: counts object from buildCardFiveElements logic }`
- "tenDeities": `{ top3: same top3 array used in buildCardTenDeities }`
- "luckCycles": `{ startAge, cycles: first 3 daYun ganZhi + startYear pairs }`
- "partnerTraits": `{ partnerElement: partnerElName from buildCardPartnerTraits logic }`
- all others: `{}` (pillars + dayMaster + strength + pattern is sufficient context)

**fetchReading(cardEl, cardType)**
Handles the full fetch-stream-render lifecycle for one card. Parameters:
- `cardEl`: the `.bazi-card` DOM element
- `cardType`: string identifier

Logic:
1. If `cardEl.dataset.readingLoaded === 'true'`, return immediately (cache guard).
2. Find the reveal button inside cardEl and hide it (`btn.style.display = 'none'`).
3. Find or create `.reading-container` inside cardEl. Clear it. Add a `<span class="reading-cursor"></span>` at the end.
4. Set `cardEl.dataset.readingLoaded = 'loading'`.
5. Build `chartSummary` via `buildChartSummary(window.currentChart, cardType)`.
6. `fetch('/api/reading', { method: 'POST', headers: {'Content-Type':'application/json'}, body: JSON.stringify({ chartSummary, cardType, language: window.currentLanguage || 'en' }) })`
7. Read response as a stream: `const reader = res.body.getReader(); const decoder = new TextDecoder();`
8. Loop: read chunks, split on `\n`, process lines starting with `data: `:
   - If line is `data: [DONE]`: remove cursor span, set `cardEl.dataset.readingLoaded = 'true'`, break.
   - If parsed JSON has `error`: show error state (see below), break.
   - If parsed JSON has `token`: append token text to a raw string accumulator, then re-render the reading container innerHTML as `renderMarkdown(accumulated) + '<span class="reading-cursor"></span>'`.
9. On fetch/network error: show error state.

**Error state:** Replace reading container content with:
```html
<div class="reading-error">
  <p>Reading unavailable. Please try again.</p>
  <button class="retry-btn" onclick="retryReading(this)">Retry</button>
</div>
```
Set `cardEl.dataset.readingLoaded = 'error'`.

**retryReading(retryBtn)**
Global function. Finds parent `.bazi-card`, resets `dataset.readingLoaded` to `''`, shows the reveal button again, clears the reading container, then calls `fetchReading(cardEl, cardEl.dataset.cardType)`.

**renderMarkdown(text)**
Minimal markdown renderer (no external library — pure JS, consistent with no-framework rule):
- `**bold**` → `<strong>bold</strong>`
- `\n\n` → `</p><p>`
- `\n- ` or `\n• ` → list items: wrap runs of `- item` lines in `<ul><li>...</li></ul>`
- Wrap entire output in `<p>...</p>`
Keep it simple — handle bold, paragraphs, and bullet lists only.

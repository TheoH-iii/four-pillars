  <action>
    Run the Vercel CLI to deploy the project:

      vercel --yes --prod

    If the Vercel CLI is not installed, install it first:

      npm install -g vercel

    If not already linked to a Vercel project, `vercel --yes` will prompt for project setup — accept defaults (no framework, output directory is root).

    Capture the deployment URL from the CLI output (format: https://*.vercel.app).

    Do NOT set ANTHROPIC_API_KEY yet — the Phase 1 function stub does not use it.
  </action>

    <automated>vercel ls 2>/dev/null | head -5</automated>
  </verify>
  <done>Vercel CLI reports a successful deployment. A production URL (https://*.vercel.app) is available.</done>

<task type="auto">
  <name>Task 2: Verify endpoints</name>
  <files></files>
  <action>
    Using the deployment URL from Task 1, run two curl checks:

    1. Front-end loads:
       curl -s -o /dev/null -w "%{http_code}" {VERCEL_URL}/
       Expected: 200

    2. Serverless function health check:
       curl -s {VERCEL_URL}/api/reading
       Expected: {"status":"ok","message":"reading endpoint ready"}

    If either check fails, inspect Vercel function logs:
       vercel logs {VERCEL_URL}

    Common issues:
    - 404 on /api/reading: check api/reading.js exports module.exports (not ES module export default)
    - 500 error: check vercel logs for syntax errors in api/reading.js
  </action>
  <verify>
    <automated>curl -s $(vercel ls --json 2>/dev/null | node -e "const d=require('fs').readFileSync('/dev/stdin','utf8'); try { const j=JSON.parse(d); console.log('https://'+j[0].url); } catch(e) { console.log(''); }") /api/reading 2>/dev/null || echo "Run: curl {VERCEL_URL}/api/reading manually"</automated>
  </verify>
  <done>curl {VERCEL_URL}/ returns HTTP 200. curl {VERCEL_URL}/api/reading returns {"status":"ok","message":"reading endpoint ready"}.</done>
